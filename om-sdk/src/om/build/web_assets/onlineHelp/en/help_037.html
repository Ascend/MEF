Restoring Default Settings

Scenarios

When the system settings are damaged during the running of the device, you can restore the default settings. The Atlas IES also supports the restoration of the minimum system. For details, see the FusionDirector Operation Guide.

  • Restoring the default settings cannot be undone. Exercise caution when performing this operation.
  • Restoring the default settings will delete all configuration information. Determine whether to retain the IP address. If the IP address is not reserved, remote access to the Atlas 500 A2 edge station may fail.
    • The default settings that can be restored include the web password, web timeout interval, password validity period, certificate expiration alarm days, NTP enabling setting, alarm masking configuration, NFS management, login rules, mounting trustlist (drive partition not mounted), NMS mode, encryption algorithm configuration, container-related configuration (enabled capability set for containerized application management), web communication certificate, and IP address (Not Retain).
    • The settings that are reserved include the IP address (Retain), white-box configuration, hostname, system asset label, system time, system time zone and region, system weak password, drive partition, and mounting trustlist (drive partition mounted).
  • After the container configuration is restored, the containers deployed before the restoration will be deleted.

Procedure

  1. Choose Settings > Restore Default Settings.

    The Basic Information operation page is displayed.

    Figure 1 Restoring default settings

  2. Select whether to retain the network IP address for the next login.

    • If yes, set IP Address to Retain.
    • If no, set IP Address to Not Retain.

      You are advised to retain the current login IP address. If the retained IP address is not in the same network segment as the default gateway, the retained IP address may fail to be used to remotely log in to the system after default settings are restored.

      If you do not retain the IP address, the IP address will be restored to the default system IP address. As a result, you may not be able to remotely access the device.

  3. Enter the password of the current administrator in the Current User Password text box.
  4. Click Save.

    In the displayed dialog box, click Confirm.

    It takes about two minutes to restore the default settings. After the default settings are restored, the system automatically refreshes the page and returns to the login page.
