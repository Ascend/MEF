证书缺少SANs导致容器镜像下发失败

问题描述

通过FusionDirector下发容器镜像时，出现“x509: certificate relies on legacy Common Name field, use SANs instead”错误提示。

可能原因

用户使用自定义对接FusionDirector的证书缺少“使用者可选名称”（SANs）字段，导致Atlas IES与FusionDirector对接失败。

用户从FusionDirector上导出的证书包含“使用者可选名称”字段，不会出现上述问题。

解决措施

  1. 重新制作证书。

    制作证书时选上“使用者可选名称”字段。

  2. 在Atlas IES的Web界面上重新进行网管注册操作，并导入新证书。
  3. 重新通过FusionDirector下发容器镜像。

    查看问题是否解决。
