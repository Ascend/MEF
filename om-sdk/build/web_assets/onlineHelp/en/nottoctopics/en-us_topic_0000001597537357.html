Container Image Fails to Be Delivered Due to Lack of SAN in the Certificate

Symptom

When a container image is delivered through FusionDirector, the error message "x509: certificate relies on legacy Common Name field, use SANs instead" is displayed.

Possible Cause

The user-defined certificate for interconnecting with FusionDirector does not contain "subject alternative name (SAN)", causing interconnection failure between the Atlas IES and FusionDirector.

If SAN is contained in the certificated exported from FusionDirector, the preceding problem does not occur.

Solution

  1. Make a new certificate.

    Include SAN when creating a certificate.

  2. Register with the NMS again on the Atlas IES WebUI and import the new certificate.
  3. Deliver the container image through FusionDirector again.

    Check whether the fault is rectified.
