Docker Service Fails to Be Started Occasionally After the System Is Forcibly Powered Off and Powered On

Symptom

After the system is forcibly powered off and then powered on, the Docker service occasionally fails to be started.

Possible Cause

After the system is forcibly powered off and then powered on, the PID of dockerd conflicts. As a result, the Docker service fails to be started.

Solution

Perform either of the following operations:

  • Log in to the background environment as the root user and run the following command to restart the Docker service:
    systemctl restart docker
  • On the Atlas IES WebUI, choose Management > Reset Control and click Restart to restart system services.
