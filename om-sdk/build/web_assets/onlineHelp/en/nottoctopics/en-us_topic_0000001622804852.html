Partition Created Successfully but Failed to Be Mounted

Symptom

A partition is successfully created on the Atlas IES WebUI, but the partition format is not ext4. At the same time, the partition fails to be mounted.

Possible Cause

After a partition is created, the system runs the partition formatting command to format the partition to ext4. However, due to system resource problems, the partition ID may fail to be obtained. As a result, the formatting fails, and the created partition is not in ext4 format. A partition in non-ext4 format will fail to be mounted.

Solution

  1. Log in to the background environment as the root user.
  2. Run the following command to format the partition to ext4:

    mke2fs -t ext4 /dev/{Name of the new partition}

  3. Return to the Atlas IES WebUI and mount the partition again.
